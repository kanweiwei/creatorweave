# 待同步文件列表 - 需求文档

## 1. 背景与问题

### 1.1 当前行为
用户通过 Python 执行代码（如生成图片、修改文件）后：
- `SyncPreviewPanel` 显示待同步的文件变更
- 用户点击"确认同步"或"取消"后
- `clearChanges()` 被调用，`pendingChanges` 变为 `null`
- **面板消失，已同步的内容无法再查看**

### 1.2 用户痛点
> "这个组件只在结束后展示，关闭后就看不到了，我需要随时查看，应该怎么做？"

**具体场景**：
- 执行 Python 代码 → 检测到文件变更
- 同步后变更清空 → 下次执行代码时才再次显示
- **无法连续查看多次文件变更状态**

---

## 2. 用户故事

### US1: 简单的待同步文件列表
**作为**频繁执行代码的用户，我想要一个简单的"待同步文件列表"视图。

**需求描述**：
- 检测到文件变更时自动显示列表
- 同步后列表自动清空
- 有新变更时自动更新显示
- 不需要复杂的历史记录功能

**验收标准**：
- [ ] 列表在检测到变更时自动显示
- [ ] 同步成功后列表自动清空
- [ ] 列表显示文件路径、类型、大小等关键信息
- [ ] 界面简洁直观

### US2: 清晰的状态指示
**作为**用户，我想要简单清晰的待同步状态，不要复杂的概念。

**验收标准**：
- [ ] 只有"待同步"和"已清空"两种状态
- [ ] 状态切换直观明显
- [ ] 不需要记住之前的同步操作

## 5. 数据模型

```typescript
/**
 * 文件变更项（简化版）
 */
interface FileChangeItem {
  path: string           // 文件路径
  type: 'add' | 'modify' | 'delete'  // 变更类型
  size?: number        // 文件大小（字节）
  mtime: number        // 修改时间
}

/**
 * 待同步文件列表状态
 */
interface PendingFileList {
  changes: FileChangeItem[]  // 当前待同步的文件列表
  timestamp: number            // 检测时间戳
}
```

## 6. UI 设计建议

### 6.1 位置建议
- **建议**：在 SyncPreviewPanel 内部优化，不要新增独立的历史记录功能
- **理由**：避免界面层级混乱，保持简单直观

### 6.2 状态显示
```
┌─────────────────────────┐
│ 📋 待同步文件      │
│                    │
│ ┌─────────────────┐  │
│ │ chart.png (新增) │  │
│ │ data.json (修改) │  │
│ │ 3 个文件待同步  │  │
│ └─────────────────┘  │
│                    │
│     [取消] [同步全部] │
│                    │
└─────────────────────────┘
```

### 6.3 行为说明
- 检测变更 → 自动显示面板
- 点击"同步" → 执行同步，完成后清空列表
- 点击"取消" → 清空列表，关闭面板
- 新的变更 → 自动更新列表
